# 简历建议

前端简历要突出你解决了什么问题、承担了什么职责、带来了什么结果。避免只堆技术名词。

## 项目表达模板

```txt
在某业务场景中，负责某模块从 0 到 1 建设。
通过某技术方案解决某问题，使某指标得到改善。
```

## 常见问题

- 项目描述只有功能列表，没有技术难点。
- 技术栈很多，但看不出掌握深度。
- 没有量化结果或业务上下文。

## 建议

每个项目准备一段 1 分钟版本和一段 3 分钟版本，分别用于简历初筛和深挖环节。
