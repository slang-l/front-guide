# Java 基础语法

Java 基础语法决定代码能否清楚表达业务逻辑。后端项目中，语法本身不难，难点在于写出边界清晰、易维护的代码。

## 变量与类型

Java 是静态类型语言，变量声明时需要明确类型。

```java
int count = 10;
String username = "front-guide";
boolean enabled = true;
```

常见类型包括基本类型和引用类型。基本类型保存值，引用类型保存对象引用。

## 方法

方法用于封装一段可复用逻辑。

```java
public int add(int a, int b) {
    return a + b;
}
```

方法签名要表达清楚输入和输出。业务方法不要返回含义模糊的 `Map`，优先使用明确的对象类型。

## 流程控制

分支和循环用于描述业务规则：

```java
if (score >= 60) {
    return "passed";
}
return "failed";
```

复杂分支要尽量拆成小方法。不要把多层 `if` 堆在一个接口方法里。

## 包与访问修饰符

包用于组织代码结构，访问修饰符用于控制可见性：

| 修饰符 | 说明 |
| --- | --- |
| `public` | 对外可见 |
| `protected` | 子类和同包可见 |
| 默认 | 同包可见 |
| `private` | 当前类内部可见 |

后端项目中，控制器、服务、仓储和领域对象通常放在不同包下，方便维护职责边界。

## 总结

Java 基础语法的目标不是炫技，而是稳定表达业务。变量类型、方法签名、分支结构和包组织越清楚，后续维护成本越低。
